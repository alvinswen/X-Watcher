# 产品概述

X-watcher 是一个**面向 Agent 的 X 平台智能信息监控服务**，通过结构化 API 为 AI Agent 提供 X 平台（Twitter）信息的采集、处理和分发能力。

## 核心能力

- **智能聚合** - 从 X（Twitter）平台抓取关注人物的动态
- **内容去重** - 自动识别和合并重复/相似内容
- **智能摘要** - 提取关键信息，生成简明摘要
- **英译中** - 将英文内容翻译为中文
- **关注列表管理** - 支持动态调整监控列表
- **运行时调度管理** - 管理员可通过 API 动态调整抓取间隔和下次触发时间
- **结构化输出** - 所有数据通过 RESTful API 以 JSON 格式输出，便于 Agent 消费

## 目标用户

**AI Agent 和开发者**
- AI Agent 需要自动化获取 X 平台信息作为上下文输入
- 开发者需要构建基于 X 平台数据的智能应用
- 需要结构化、经过清洗的高质量数据源

## 核心使用场景

1. **Agent 自动化信息采集**
   - Agent 通过 Feed API 定时增量拉取处理后的推文数据
   - 获取去重、摘要、翻译一站式处理的结构化信息

2. **动态监控配置**
   - 通过 API 动态调整关注列表
   - 支持按需切换监控重点

3. **信息分发服务**
   - 为下游 Agent 或应用提供标准化的信息流
   - 支持增量拉取，避免重复处理

## 价值主张

与通用信息聚合器不同，X-watcher 的核心差异在于：

- **Agent 友好 > 人机交互**
  - RESTful API + JSON 输出，面向程序化消费
  - 增量拉取机制，适配 Agent 的定时调度模式

- **深度处理 > 简单聚合**
  - 去重、摘要、翻译一站式完成
  - 减少信息噪音，提高信噪比

- **服务化 > 工具化**
  - 作为独立服务运行，支持多 Agent 并发调用
  - 标准化接口，可被任意 Agent 框架集成

## 未来扩展方向

- **数据源扩展** - 从 X 平台扩展到 Newsletter、RSS 等更多来源
- **MCP 集成** - 提供 Model Context Protocol 接口，直接对接 LLM 应用
- **Webhook 推送** - 支持主动推送新信息到 Agent 端
